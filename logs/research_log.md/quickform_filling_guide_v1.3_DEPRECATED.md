# **QuickForm v1.3 填寫指南**

**對應工具：** CKV\_QuickForm\_v1\_3.html  
 **依據來源：** schema consistency review \+ 前三筆紀錄 QC 回饋  
 **日期：** 2026-03-10

---

## **Step 1 — 地點**

### **場域類型 site\_type**

**判斷原則：** 用分類，不用描繪。問自己：「如果我在 Google Maps 或建築資料裡搜尋這個地方，它會被歸為什麼？」

| 常見錯誤 | 正確做法 |
| ----- | ----- |
| 桂離宮標為 `zen_temple_garden` | 桂離宮是皇室離宮，應為 `imperial_garden` |
| 二年坂標為 `urban_public_space` | 二年坂的主體是歷史階梯街區，`urban_public_space` 過於寬泛 |

**現有選項：** zen\_temple\_garden / imperial\_garden / urban\_public\_space / agricultural\_terroir / winery\_vineyard / immersive\_overnight / natural\_landscape / commercial\_district / interview\_accompanied / other

**如果猶豫：** 寧可選更窄的分類。如果現有選項都不對，選 `other` 並在 research\_finding 裡說明。

### **同行者 companion**

**統一格式：** 使用英文小寫。

| 值 | 說明 |
| ----- | ----- |
| solo | 獨自 |
| husband | 與老公 |
| friend | 與朋友 |
| group | 團體 |

不要混用「老公」「husband」「Solo」。選一個語言，保持一致。

### **天氣 weather**

**統一格式：** 使用中文單字或短詞。

| 值 | 說明 |
| ----- | ----- |
| 晴 | 晴天 |
| 陰 | 陰天 |
| 雨 | 下雨 |
| 雪 | 下雪 |
| 多雲 | 多雲但不算陰 |

三筆紀錄都用了中文（晴、陰、雨），維持這個慣例。

## **Step 2 — 評分**

### **每個維度的 field note**

**核心規則：分數越高，筆記越具體。**

| 分數範圍 | 最低要求 |
| ----- | ----- |
| 0–4 | 一句話說明為什麼低 |
| 4.5–7 | 至少一個具體觀察 \+ 判斷理由 |
| 7.5–10 | 至少兩個可觀測的感官細節，不能只有斷言 |

**反面教材：**

「無庸置疑，這個項目是絕對的高分」（KYT-20260118-A, Vg=9）

這是評價，不是觀察。Vg=9 需要描述具體的幾何事件——視線怎麼被引導、空間如何轉折、比例關係是什麼。

**正面教材：**

「二年坂三年坂的階梯步道形成一個非常特殊的幾何構圖，主要是因為看五重塔的角度因為階梯高低而出現幾何 anchor 的變化。」（KYT-20260226-A, Vg=8）

這裡有機制（階梯高低）、有對象（五重塔）、有效果（角度變化）。

### **各維度填寫提示**

**Vt — 傳統文化密度：** 寫你在現場看到的具體文化元素（建築、儀式、材質、歷史痕跡），不要寫你對歷史的詮釋。

**Vl — 光線質感：** 描述光在物體表面上實際做了什麼——影子的形狀、反射的位置、材質在光下的質感。不要推測設計者的意圖。如果當時是晴天，描述晴天的光具體讓你看到什麼。

**Vr — 空間克制度：** 只寫節奏和律動。人怎麼移動、速度快慢、能不能自由停留、空間引導你往哪走。不要在這裡寫情緒反應——那是 Ve 的工作。

**Vg — 空間幾何結構感：** 描述幾何事件——視線通道、空間比例、framing 關係、動線幾何。幾何描述應該集中在這裡，不要散落到 Vr 或 Ve 的欄位。

**Ve — 時間感：** 寫你的情緒反應和時間感受。如果你在 Vr 已經寫了被控制的節奏，這裡寫的是那個節奏讓你「感覺」到什麼——不悅、沉浸、解脫、無聊。另外，**標注方向**：

| 方向標記 | 用法 |
| ----- | ----- |
| 正向低強度 | 舒適但沒有被打動（例：二年坂 Ve=2.5） |
| 負向低強度 | 輕微不適或想離開（例：LaLaport Ve=2.5） |
| 正向高強度 | 深度共鳴或沉浸 |
| 負向高強度 | 強烈不悅或被壓迫 |
| 混合/矛盾 | 同時有正向和負向（例：桂離宮 Ve=5.5，讚嘆 \+ 被控制感） |

在 field note 開頭或結尾加一句方向標記即可。

## **Step 3 — 標註**

### **Vt 子類型**

| 標籤 | 適用場景 |
| ----- | ----- |
| architectural\_heritage | 歷史建築群為主體（寺院、町家、離宮） |
| agricultural\_terroir | 農業文化景觀（茶園、梯田、酒莊） |
| biographical\_founder | 文化密度來自特定人物故事 |
| intangible\_practice | 非物質文化實踐（茶道、祭典現場） |
| corporate\_constructed | 企業建構的文化符號 |
| non\_intervention\_natural | 非人為介入的自然文化 |

### **Vl 光源類型**

直接對應現場光源，不需要推測。

| 標籤 | 判斷基準 |
| ----- | ----- |
| natural\_diffused | 戶外陰天或間接自然光 |
| natural\_direct | 戶外晴天直射 |
| artificial\_warm | 室內暖色人工光 |
| artificial\_neutral | 室內冷白或中性人工光 |
| mixed | 自然光與人工光混合 |
| candlelight\_lantern | 燭光或燈籠 |

**注意：** KYT-20260118-A 在晴天戶外卻標為 `natural_diffused`。晴天直射應為 `natural_direct`。如果庭園內有遮蔭產生漫射效果，在 field note 裡說明。

### **Vr 來源**

**判斷原則：** 選擇造成節奏的主要原因，只選一個。如果多個因素共存，選最主導的。

| 標籤 | 主導因素 | 典型場景 |
| ----- | ----- | ----- |
| architectural\_design | 空間本身的設計決定你怎麼移動 | 迴廊、階梯步道、庭園動線 |
| social\_protocol | 社會規範決定行為節奏 | 茶室、宗教場所 |
| invited\_openness | 空間邀請你自由進入和停留 | 開放廣場、公園 |
| cognitive\_compression | 資訊或感官過載壓縮思考空間 | 商業空間、觀光熱區 |
| existential\_permission | 空間允許你暫停存在節奏 | 深山、海邊、冥想空間 |
| guided\_tour\_managed | 導覽制度決定移動節奏 | 桂離宮、需預約的場域 |

**桂離宮的案例：** 原本標為 `architectural_design`，但 field note 寫的主要是導覽制度限制。如果無法自由停留的原因是導覽規則而非建築本身，應選 `guided_tour_managed`。

### **Vg 幾何子類型**

**判斷原則：** 問自己空間的幾何性格是什麼——是包圍你，還是向外展開？是有明確框架，還是開放無界？

| 標籤 | 判斷基準 |
| ----- | ----- |
| enclosure\_deep | 深度包圍感（洞窟、深廊） |
| enclosure\_shallow | 淺層包圍（茶室、小庭院） |
| open\_framed | 開放但有明確框架引導視線（桂離宮的月見台） |
| open\_unframed | 開放且無引導、視野自由 |
| layered\_threshold | 多層門檻結構（鳥居序列、寺院山門層遞） |
| void\_centred | 以虛空為中心（枯山水、中庭） |
| geometric\_strict | 嚴謹幾何秩序（對稱、網格） |
| geometric\_organic | 有幾何感但帶有機曲線 |

**注意：** 如果 field note 裡寫了「anchor」「framing」「引導視線」，那就不是 `open_unframed`——有 anchor 就有 frame。

### **Ve 啟動機制**

**判斷原則：** 問自己情緒是被什麼觸發的——是空間本身、你自己的投射、還是外來的敘事？

| 標籤 | 判斷基準 | 怎麼確認 |
| ----- | ----- | ----- |
| intrinsic\_spatial | 空間本身直接產生情緒 | 不需要任何背景知識就能感受到 |
| self\_time\_projection | 你自己的時間投射（想像過去/未來） | 情緒來自你的聯想，不是空間給的 |
| external\_narrative\_human | 他人敘事介入（導覽員、同行者、書籍） | 可以指出是誰說了什麼 |
| external\_narrative\_AI | AI 即時敘事介入 | 現場使用了 AI 工具查詢資訊 |
| preloaded\_knowledge | 預先具備的知識被現場激活 | 你已經知道的事情在現場突然變得有感 |
| none | 沒有明顯情緒觸發 | 感官平淡，無特殊情緒事件 |

**桂離宮的案例：** 原本標為 `self_time_projection`，但情緒軌跡明確記載「求證 Google 的 AI 模式」後情緒轉向。應標為 `external_narrative_AI`。如果有兩層觸發（先自我投射、後 AI 確認），field note 裡標記兩階段，trigger 選最終主導情緒方向的那個。

### **觀察者狀態標記**

**sensory\_saturation\_effect：** 勾選時，在相關維度的 field note 裡加一句說明哪些維度受影響、方向如何。例如：「長期暴露同類型商業空間，Ve 感知已習慣化，可能造成 Ve 低估。」

**ai\_collaboration：** 勾選時，在受影響的維度 field note 裡標記 AI 介入的時間點。例如：「離場前查詢 AI 確認歷史背景，此後 Ve 方向從審美距離轉為共感。」

## **Step 4 — 情緒**

### **入場狀態 entry\_state**

寫你走進場域那一刻的心理狀態。不需要文學性——寫清楚就好。

| 好的範例 | 壞的範例 |
| ----- | ----- |
| 帶著研究者視角進場，預期低，保持觀察距離 | 今天心情不錯 |
| 午餐後散步回家，沒有特別期待 | 感覺很好 |

### **轉折點 turning\_point**

如果沒有轉折，留空。QuickForm 會輸出 `"none"`。

如果有轉折，分清楚：

1. **你在轉折前觀察到什麼**（observation 層）  
2. **是什麼事件造成了轉折**（trigger）  
3. **轉折後你的感受如何改變**（effect）

如果使用了 AI，標記清楚哪些句子是 AI 介入前的觀察，哪些是介入後的詮釋。可以用簡單標記：

* `[現場觀察]` 我覺得住在這裡的人應該很無聊  
* `[查詢AI後]` 求證 Google AI，確認這確實是精緻孤獨的空間

### **離場狀態 exit\_state**

**規則：只寫這次造訪的直接感受，不要寫跨情境的結論。**

| 可以 | 不可以 |
| ----- | ----- |
| 這次走進去，我感覺到一份無奈 | 那份無奈就是 400 年前那位親王生活的主旋律 |
| 離場時有輕微的感官解脫感 | 去掉櫻花之後，結構更清楚 |

第二欄的句子是有效的 insight，但它們超出了單筆紀錄的證據範圍——它們預設了對照條件（有櫻花 vs 沒有）或歷史心理判斷。這類句子適合放在 key\_insight（標記為假設），不適合放在 exit\_state（應為直接感受）。

### **核心洞察 key\_insight**

這裡可以寫你最精練的詮釋。但如果它是一個假設而不是一個觀察結論，標記出來。

| 已驗證的觀察 | 未驗證的假設 |
| ----- | ----- |
| 建築驅動型商場，不是氛圍驅動型商場 | 極度無聊的人，用極致的才華蓋出逃避現實的空間（需歷史文獻驗證） |

## **Step 5 — 備註**

### **研究發現 research\_finding**

**核心問題：這筆紀錄對 dataset 的具體貢獻是什麼？**

| 好的 finding | 壞的 finding |
| ----- | ----- |
| 第一筆台北低端錨點。驗證了 SDIndex 核心邏輯：單一維度高不足以拉升整體分數⋯⋯ | AI 即時的內容補充有可能改變人類感官 |
| 在無花無葉的冬季條件下，二年坂的空間結構仍可獨立產生 Vg=8 的構圖強度 | 沒有櫻花與楓葉的二年坂，京都的空間結構仍然成立嗎？ |

左欄是 finding——有具體數據、有條件限定、有結論。右欄第一個是通則假設（可以套用在任何場域），第二個是 question（不是 finding）。

### **最佳採樣條件 ideal\_condition**

寫出讓這個空間達到最高 SDIndex 的客觀條件。要具體到可被其他研究者重現。

| 好 | 不夠好 |
| ----- | ----- |
| 冬天週間大太陽，人潮稀少 | 最好的時候去 |
| 京都旅遊淡季的清晨與深夜 | 沒有人的時候 |
| 楓葉季 | 秋天 |

## **填寫前的最後檢查清單**

在按下「下載 .json」之前，花 30 秒過一遍：

* \[ \] site\_type 是分類，不是描繪？  
* \[ \] 每個 ≥ 8 分的維度都有至少兩個具體觀察？  
* \[ \] 幾何描述在 Vg note 裡，不是散落在 Vr 或 Ve？  
* \[ \] 情緒反應在 Ve note 裡，不是混在 Vr？  
* \[ \] Ve note 有標注方向（正向/負向/混合）？  
* \[ \] 如果勾了 ai\_collaboration，相關 field note 有標記 AI 介入時間點？  
* \[ \] 如果勾了 sensory\_saturation\_effect，有說明影響哪些維度？  
* \[ \] research\_finding 是這筆紀錄的具體貢獻，不是通則假設或問題？  
* \[ \] companion 和 weather 格式一致？
