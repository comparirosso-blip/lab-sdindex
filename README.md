# Lab_SDIndex

A research repository for SDIndex protocol development, structured field observation, and future Graph RAG alignment.

## Overview

Lab_SDIndex is a research-oriented repository for building, documenting, and testing the **SDIndex** framework.

This repository is not a generic note archive.  
It is designed as a **protocol lab** for:

- structured sensory field observation
- schema-based recording consistency
- SDIndex scoring logic
- gold sample accumulation
- future interoperability with Graph RAG / knowledge graph systems

The core idea is simple:

**Human field observation first. Structured semantic consistency second. Graph integration later.**

This repo exists to preserve that order.

## Purpose

The purpose of this repository is to provide a stable foundation for:

1. documenting the SDIndex protocol
2. storing structured observation records
3. refining schema consistency across records
4. identifying high-quality gold samples
5. preparing future machine-readable links for Graph RAG systems

This is a human-led research system.  
AI may assist interpretation, but the primary source of truth is **direct human observation**.

## Research Position

SDIndex is based on the assumption that many meaningful spatial, sensory, and emotional experiences cannot be captured by conventional tourism description, rating systems, or generic metadata.

This repository treats field observation as a form of **structured epistemic labor**.

The project stands on three core capabilities:

- **Observation**  
  Careful, embodied, first-hand noticing of space, atmosphere, rhythm, contrast, and affective shifts.

- **Schema Thinking**  
  Translating subjective experience into a stable, reusable, structured recording system.

- **Consistency**  
  Applying the same protocol across different sites so that comparison, accumulation, and future computation become possible.

## Repository Scope

This repository may include:

- protocol documents
- schema definitions
- observation templates
- finalized JSON records
- review notes
- methodology notes
- gold sample selection logic
- Graph RAG preparation notes

This repository does **not** assume that every record is immediately suitable for downstream AI use.

Only records with sufficient clarity, internal consistency, and semantic stability should later be considered for:

- graph ingestion
- ontology mapping
- retrieval experiments
- benchmark evaluation
- gold sample reference sets

## Guiding Principle

Do not over-engineer before data exists.

The system should grow in this order:

1. protocol clarity
2. field data accumulation
3. consistency review
4. gold sample identification
5. graph alignment
6. retrieval and reasoning experiments

In other words:

**Collect first. Stabilize second. Connect later.**
